БИЛЕТ 1.
1. Символы для именования элементов текста программы (идентификаторов) в Java:
   - Буквы (латинские a-z, A-Z, символ подчеркивания `_` и знак доллара `$`).
   - Цифры (0-9), но не на первой позиции.
   - Имена чувствительны к регистру (например, `myVar` и `myvar` - разные имена).
   - Нельзя использовать зарезервированные ключевые слова (например, `class`, `int`, `for`).

   Примеры именования:
   - Корректные: `myVariable`, `_count`, `$value`, `number1`, `MyClass`.
   - Некорректные: `1number` (начинается с цифры), `my-var` (содержит дефис), `class` (ключевое слово).

2. Типы комментариев и символы для их обозначения:
   - Однострочный комментарий: `//`
   - Многострочный комментарий: `/* ... */`
   - Комментарий для документации (Javadoc): `/** ... */`

   Примеры комментариев:
   - Однострочный: `// Это однострочный комментарий`
   - Многострочный:
        /*
           Это многострочный
           комментарий
        */
   - Javadoc:
        /**
           Этот метод возвращает сумму двух чисел.
           @param a первое число
           @param b второе число
           @return сумма a и b
        */

3. Префикс для обозначения чисел 16-й системы счисления: `0x` или `0X`.

   Примеры чисел в 16-й системе:
   - `0x1A` (десятичное 26)
   - `0xFF` (десятичное 255)
   - `0xCAFEBABE` (десятичное 3405691582)

БИЛЕТ 2.
1. Названия и параметры целочисленных типов со знаком и вещественных типов в Java:
   Целочисленные типы (все со знаком):
   - `byte`: 8 бит, диапазон от -128 до 127.
   - `short`: 16 бит, диапазон от -32,768 до 32,767.
   - `int`: 32 бита, диапазон от -2,147,483,648 до 2,147,483,647.
   - `long`: 64 бита, диапазон от -9,223,372,036,854,775,808 до 9,223,372,036,854,775,807.
   Вещественные типы (с плавающей точкой):
   - `float`: 32 бита, приблизительный диапазон ±3.4e-38 до ±3.4e+38, около 7 значащих цифр.
   - `double`: 64 бита, приблизительный диапазон ±4.9e-324 до ±1.8e+308, около 15-16 значащих цифр.

2. Назначение методов классов-оберток:
   Эти классы (`Byte`, `Short`, `Integer`, `Long`, `Float`, `Double`) используются для "обертывания" примитивных типов в объекты.
   Основные методы и их назначение:
   - `valueOf()`: статический метод, преобразует строку или примитив в объект соответствующего класса-обертки.
   - `parseXxx()`: статический метод, преобразует строку в примитив (например, `Integer.parseInt("123")`).
   - `toString()`: преобразует объект или примитив в строку.
   - `xxxValue()`: преобразует объект-обертку в примитив (например, `intValue()`, `doubleValue()`).
   - `compareTo()`: используется для сравнения двух объектов-оберток.
   - `MAX_VALUE`, `MIN_VALUE`: статические поля, содержащие максимальное и минимальное значения для данного типа.

БИЛЕТ 3.
1. Название символьного типа и его назначение:
   - `char`: 16-битный беззнаковый тип данных, представляющий отдельный символ в кодировке Unicode.
   - Назначение: хранение и обработка одиночных символов (буквы, цифры, знаки препинания, управляющие символы).

   Характеристика кодировок:
   - Unicode: 16-битная кодировка (в Java), позволяет представить символы большинства письменных систем мира (до 65,536 символов, хотя с суррогатными парами охватывает более 1 млн).
   - ASCII: 7-битная кодировка (128 символов), является подмножеством Unicode (первые 128 кодов Unicode соответствуют ASCII).

   Название логического типа и его назначение:
   - `boolean`: может принимать только два значения: `true` или `false`.
   - Назначение: используется в условных выражениях (например, в операторах `if`, `while`) и для хранения результата логических операций.

   Служебные слова для логических значений:
   - Истина: `true`
   - Ложь: `false`

БИЛЕТ 4.
1. Назначение массивов и варианты их объявления:
   - Массив - это структура данных для хранения фиксированного количества элементов одного типа.
   - Одномерный массив: используется для хранения линейной последовательности элементов. Объявление:
     `тип[] имяПеременной;` или `тип имяПеременной[];`
     `int[] numbers;`
     `int numbers[];`
   - Двумерный массив: используется для хранения табличных данных (матриц) или массивов массивов. Объявление:
     `тип[][] имяПеременной;` или `тип имяПеременной[][];`
     `int[][] matrix;`
     `int matrix[][];`

   Примеры с инициализацией:
   - `int[] arr = new int[10];`
   - `int[] arr = {1, 2, 3};`
   - `int[][] matrix = new int[5][5];`
   - `int[][] matrix = {{1,2}, {3,4}};`

БИЛЕТ 5.
1. Обозначение и назначение основных операций в Java:
   - Инкремент (`++`) и декремент (`--`): увеличивают/уменьшают значение переменной на 1. Бывают префиксными (сначала меняют значение, затем используют) и постфиксными (сначала используют, затем меняют).
   - Арифметические операции: `+`, `-`, `*`, `/`, `%` (остаток от деления).
   - Операции отношения: `==`, `!=`, `<`, `>`, `<=`, `>=`. Результат - `boolean`.
   - Логические операции: `&&` (И), `||` (ИЛИ), `!` (НЕ), `^` (исключающее ИЛИ). Используются с `boolean`.
   - Поразрядные операции: `&` (И), `|` (ИЛИ), `^` (исключающее ИЛИ), `~` (НЕ), `<<` (сдвиг влево), `>>` (сдвиг вправо), `>>>` (беззнаковый сдвиг вправо). Работают с битами чисел.
   - Операция присваивания: `=` (присваивает значение правой части левой части).
   - Операции составного присваивания: `+=`, `-=`, `*=`, `/=`, `%=`, `&=`, `|=`, `^=`, `<<=`, `>>=`, `>>>=`. Выполняют операцию и присваивают результат.
   - Операция указания типа (приведение): `(тип)`. Используется для явного преобразования типов.
   - Операция `new`: используется для создания нового объекта (выделения памяти и вызова конструктора).

БИЛЕТ 6.
1. Операторы цикла в общем виде, их назначение и порядок выполнения:
   - `for` (цикл со счетчиком):
     `for (инициализация; условие; итерация) { тело_цикла; }`
     Порядок: 1) Инициализация (выполняется один раз). 2) Проверка условия. 3) Если условие `true` -> выполняется тело цикла. 4) Выполняется итерация. 5) Переход к шагу 2.

   - `while` (цикл с предусловием):
     `while (условие) { тело_цикла; }`
     Порядок: 1) Проверка условия. 2) Если условие `true` -> выполняется тело цикла. 3) Переход к шагу 1.

   - `do-while` (цикл с постусловием):
     `do { тело_цикла; } while (условие);`
     Порядок: 1) Тело цикла выполняется один раз. 2) Проверка условия. 3) Если условие `true` -> переход к шагу 1.

   - `continue`: оператор, используемый внутри тела цикла. При его выполнении текущая итерация цикла немедленно завершается, и управление передается на проверку условия цикла (или на шаг итерации в цикле `for`).

БИЛЕТ 7.
1. Условные операторы в общем виде, их назначение и порядок выполнения:
   - `if` (условный оператор):
     `if (условие) { блок_кода1 } [else { блок_кода2 }]`
     Порядок: 1) Вычисляется условие. 2) Если условие `true`, выполняется `блок_кода1`. 3) Если условие `false` и есть `else`, выполняется `блок_кода2`.

   - `switch` (оператор множественного выбора):
     `switch (выражение) { case значение1: блок_кода1; break; case значение2: блок_кода2; break; ... default: блок_кода_по_умолчанию; }`
     Порядок: 1) Вычисляется `выражение`. 2) Значение сравнивается со значениями в `case` (используется `equals` для строк и `==` для примитивов). 3) При совпадении выполняется соответствующий блок кода до оператора `break`. 4) Если совпадений нет и есть `default`, выполняется его блок.

БИЛЕТ 8.
1. Объявление класса в общем виде:
   `[модификаторы] class ИмяКласса [extends РодительскийКласс] [implements Интерфейсы] { тело_класса }`

   Назначение полей класса:
   Поля (или переменные экземпляра) - это данные, которые определяют состояние объекта класса. Каждый объект класса имеет свои собственные копии этих полей.

   Назначение спецификаторов доступа:
   - `public`: элемент (поле, метод, класс) доступен из любого другого класса.
   - `private`: элемент доступен только внутри того класса, в котором он объявлен.
   - `protected`: элемент доступен внутри пакета и в подклассах (даже если они находятся в других пакетах).
   (Если спецификатор не указан, доступ по умолчанию (package-private) - элемент доступен только внутри своего пакета).

БИЛЕТ 9.
1. Объявление метода в общем виде:
   `[модификаторы] типВозврата имяМетода( [параметры] ) [throws Исключения] { тело_метода }`

   Назначение параметров, локальных и глобальных переменных:
   - Параметры метода: переменные, перечисленные в скобках при объявлении метода. Они принимают значения аргументов, переданных при вызове метода.
   - Локальные переменные: переменные, объявленные внутри метода. Они существуют только во время выполнения метода и доступны только внутри него.
   - Глобальные переменные (поля класса): объявляются вне методов, внутри класса. Доступны для всех методов класса (с учетом модификаторов доступа).

   Назначение аргументов метода:
   Аргументы - это фактические значения, которые передаются методу при его вызове. Они присваиваются соответствующим параметрам метода.

   Оператор вызова метода в общем виде:
   `имяОбъекта.имяМетода(аргументы);` (для статических методов: `ИмяКласса.имяМетода(аргументы);`)
   Назначение: выполняет код, определенный в методе, с переданными аргументами.

   Назначение оператора `return`:
   Используется для:
   1) Завершения выполнения метода.
   2) Возврата значения вызывающему коду (если тип возврата метода не `void`).

   Сигнатура метода в общем виде:
   `имяМетода(типы_параметров)`
   Сигнатура уникально идентифицирует метод в классе и используется для перегрузки (overloading) методов.

БИЛЕТ 10.
1. Объявление конструктора класса в общем виде:
   `[модификаторы] ИмяКласса( [параметры] ) { тело_конструктора }`
   Особенности: не имеет типа возврата (даже `void`), его имя должно совпадать с именем класса.

   Конструкторы класса с параметрами и без параметров:
   - Конструктор без параметров (конструктор по умолчанию): не принимает аргументов. Он инициализирует поля значениями по умолчанию (например, 0 для чисел, `null` для объектов).
   - Конструктор с параметрами: принимает аргументы, позволяя инициализировать состояние объекта при его создании. Используется для перегрузки конструкторов.

БИЛЕТ 11.
1. Назначение спецификатора static:
   - `static` означает, что член класса (поле или метод) принадлежит самому классу, а не его экземплярам (объектам).
   - Статические поля существуют в единственном экземпляре для всех объектов класса.
   - Статические методы могут быть вызваны без создания объекта класса.
   - Статические методы могут обращаться только к другим статическим членам класса.

   Процесс создания объекта:
   `ИмяКласса имяОбъекта = new ИмяКласса(аргументы);`
   Процесс: 1) Оператор `new` выделяет память для объекта. 2) Вызывается конструктор класса для инициализации объекта. 3) Созданный объект возвращается и присваивается переменной.

   Назначение метода `main`:
   Метод `public static void main(String[] args)` - это точка входа в любую Java-программу. С него начинается выполнение приложения.

   Назначение служебного слова `this`:
   `this` - это ссылка на текущий объект. Используется внутри конструкторов и методов экземпляра:
   - Для различения полей класса и параметров с одинаковыми именами (`this.поле = параметр`).
   - Для вызова другого конструктора этого же класса (`this(аргументы)`).
   - Для передачи текущего объекта в качестве аргумента другому методу.

БИЛЕТ 12.
1. Понятие наследования классов:
   Наследование - это механизм ООП, позволяющий создать новый класс (подкласс) на основе существующего (суперкласса). Подкласс наследует поля и методы суперкласса и может добавлять свои собственные.

   Переопределенный метод (Overriding):
   Это метод в подклассе, который имеет ту же сигнатуру (имя, параметры) и возвращаемый тип, что и метод в суперклассе. Он заменяет (переопределяет) поведение родительского метода для объектов подкласса.

   Абстрактный метод и класс:
   - Абстрактный метод: это метод, объявленный без реализации (только сигнатура). Он должен быть реализован в первом конкретном (неабстрактном) подклассе.
   - Абстрактный класс: это класс, который не может быть инстанцирован (нельзя создать его объект). Он может содержать как абстрактные методы, так и конкретные.
   Служебное слово для обозначения: `abstract`.

   Назначение спецификатора `final`:
   - Для класса: предотвращает наследование от этого класса.
   - Для метода: предотвращает переопределение этого метода в подклассах.
   - Для переменной: делает ее константой (значение не может быть изменено после инициализации).

БИЛЕТ 13.
1. Назначение методов класса String и их параметров:
   Класс `String` представляет неизменяемые строки.

   Основные методы:
   - `length()`: возвращает количество символов в строке. Параметров нет.
   - `charAt(int index)`: возвращает символ по указанному индексу. Параметр: `index`.
   - `substring(int beginIndex, int endIndex)`: возвращает новую строку, которая является подстрокой данной. Параметры: начало и конец (не включая) подстроки.
   - `indexOf(String str)`: возвращает индекс первого вхождения подстроки `str`. Параметр: искомая подстрока.
   - `toLowerCase()` / `toUpperCase()`: возвращает строку в нижнем/верхнем регистре. Параметров нет.
   - `trim()`: возвращает строку с удаленными начальными и конечными пробелами. Параметров нет.
   - `replace(char oldChar, char newChar)`: заменяет все вхождения одного символа на другой. Параметры: старый и новый символы.
   - `split(String regex)`: разбивает строку на массив подстрок по указанному разделителю. Параметр: регулярное выражение.
   - `equals(Object obj)`: сравнивает строки на равенство (с учетом регистра). Параметр: объект для сравнения.
   - `equalsIgnoreCase(String anotherString)`: сравнивает строки без учета регистра. Параметр: строка для сравнения.

БИЛЕТ 14. (Вопрос 2 - программа с GUI, поэтому только теория по RandomAccessFile)
1. Назначение конструкторов и методов класса RandomAccessFile и их параметров:
   Класс `RandomAccessFile` позволяет читать и записывать данные в файл в произвольном порядке (не последовательно). Он работает с файлами как с большим массивом байтов.

   Конструкторы:
   - `RandomAccessFile(String name, String mode)`: создает экземпляр для файла с именем `name` в режиме доступа `mode`.
   - `RandomAccessFile(File file, String mode)`: создает экземпляр для объекта `File` `file` в режиме доступа `mode`.
   Параметр `mode`: может быть `"r"` (только чтение), `"rw"` (чтение и запись), `"rws"` (чтение, запись с синхронизацией метаданных) и `"rwd"` (чтение, запись с синхронизацией данных).

   Основные методы:
   - `seek(long pos)`: устанавливает позицию указателя файла для чтения/записи на байт `pos`.
   - `getFilePointer()`: возвращает текущую позицию указателя файла.
   - `length()`: возвращает длину файла в байтах.
   - `readByte()`, `readInt()`, `readUTF()`, `read(byte[] b)` и т.д.: читают данные различных типов из файла, начиная с текущей позиции указателя.
   - `writeByte(int v)`, `writeInt(int v)`, `writeUTF(String str)`, `write(byte[] b)` и т.д.: записывают данные различных типов в файл, начиная с текущей позиции указателя.
   - `close()`: закрывает файл и освобождает ресурсы.

БИЛЕТ 19.
1. Назначение полей и методов класса Math и их параметров:
   Класс `Math` предоставляет набор статических методов для выполнения основных математических операций.

   Поля:
   - `Math.PI`: константа, значение числа Пи (приблизительно 3.14159).
   - `Math.E`: константа, основание натурального логарифма (приблизительно 2.71828).

   Основные методы:
   - `abs(a)`: возвращает абсолютное значение числа `a` (параметр: `int`, `long`, `float` или `double`).
   - `ceil(a)`: возвращает наименьшее целое число, большее или равное `a` (параметр: `double`).
   - `floor(a)`: возвращает наибольшее целое число, меньшее или равное `a` (параметр: `double`).
   - `round(a)`: возвращает ближайшее целое число к `a` (параметр: `float` -> `int`, `double` -> `long`).
   - `max(a, b)`: возвращает большее из двух чисел `a` и `b` (параметры: `int`, `long`, `float` или `double`).
   - `min(a, b)`: возвращает меньшее из двух чисел `a` и `b` (параметры: `int`, `long`, `float` или `double`).
   - `pow(a, b)`: возвращает `a` в степени `b` (параметры: `double`).
   - `sqrt(a)`: возвращает квадратный корень из `a` (параметр: `double`).
   - `random()`: возвращает псевдослучайное число типа `double` от 0.0 до 1.0 (параметров нет).
   - `sin(a)`, `cos(a)`, `tan(a)`: возвращают тригонометрические функции угла `a` (в радианах). Параметр: `double`.
